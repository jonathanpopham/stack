the ghostie stack

AI you own — that remembers, stays local, and proves its own work.

Today's AI forgets you when the tab closes, sends your code to someone else's servers, and asks you to trust an answer it can't justify. The ghostie stack is the deliberate opposite: a set of small, local-first tools that persist your context, run entirely on your own machine, and verify their own output.

what it brings

Three things mainstream AI structurally can't give you.

Every hosted assistant is amnesiac, cloud-bound, and unverifiable — because that's the business model. This stack is the deliberate inverse of all three.

01

It remembers.anti-amnesia

Mainstream assistants forget you the instant the session closes. The ghostie stack keeps a memory vault and captures every session, so context compounds instead of resetting. What one agent learns, the next already knows — across days, across tools.

02

It's sovereign.anti-cloud

Local-first, airgap-safe, zero external dependencies, no telemetry. Your code and your context never leave your machine. In a market that rents intelligence back to you by the token, this is a stack you own outright — and can run with the network unplugged.

03

It proves its work.anti-black-box

It doesn't ask you to trust its confidence. Deterministic analysis, self-verifying certificates, and a discipline of making a test fail before trusting it — the output arrives with evidence attached, and it stays honest about what it cannot prove rather than bluffing past it.

Working pieces you can read right now.

The public repositories below are real — clone them, read the tests, read the gate. Zero external dependencies, deterministic, and gated in CI.

Lockstep

public

Behavioral-equivalence verification for code migration: runs the old and new programs over the same inputs, compares them at the observable boundary, and issues a deterministic, hash-chained certificate — including what it could not prove.

Rust · std-only, zero deps · #![forbid(unsafe)] · 35 tests · self-audited
github.com/jonathanpopham/lockstep

Ontoloom

public

An airgapped, dependency-free ontology & knowledge-graph builder with a visual browser GUI served on localhost, exporting Neo4j-compatible formats. No cloud, no telemetry, no packages.

Rust · std-only · browser SPA on loopback · Neo4j-compatible exports
github.com/jonathanpopham/ontoloom

ghostie

public

The memory layer: a persistent assistant with a memory vault and session capture, so context survives across sessions and agents instead of resetting to zero every time.

persistent memory · session capture · terminal-native
github.com/jonathanpopham/ghostie

TrailTracker

private

The understanding engine at the core of the stack: a deterministic, airgap-safe code-analysis engine with ~60 hand-written language frontends (no tree-sitter), seven graph ontologies over a codebase, and byte-stable output. The most mature piece of the whole stack.

Rust · std-only, zero deps · 62 languages · 2,354 tests · ~200k LOC · byte-stable
source private — walkthrough on request

Honest status: the stack is half-built by design. Lockstep, Ontoloom, and ghostie are public and working; TrailTracker is the mature private engine at its core; heinzel is in progress. The vision on this page is the direction — these are the parts already standing.
